## OpenClaw 메모리의 진실의 원천: 파일

OpenClaw는 복잡한 벡터 데이터베이스에 의존하는 대신, **마크다운 파일**을 '진실의 원천(Source of Truth)'으로 삼는 독특한 철학을 가지고 있습니다. 이는 AI의 기억이 단순히 서버 어딘가에 저장되는 것이 아니라, 사용자가 직접 제어하고 편집할 수 있는 문서 형태로 존재해야 함을 의미합니다.

### 1. 2레이어 메모리 구조

OpenClaw의 기억은 크게 두 가지 층으로 나뉩니다.

*   **장기 기억 (`MEMORY.md`)**: 핵심 원칙, 중요한 결정 사항, 사용자 선호도 등을 담습니다. 보안을 위해 1:1 세션(Main Session)에서만 로드되어 개인 정보 유출을 방지합니다.
*   **단기 로그 (`memory/YYYY-MM-DD.md`)**: 매일 발생하는 작업의 맥락과 세부 사항을 기록합니다.

이러한 분리 구조는 AI가 불필요한 노이즈에 휘둘리지 않으면서도, 필요한 핵심 맥락을 놓치지 않게 도와줍니다.

### 2. 하이브리드 검색과 로컬 성능

OpenClaw는 단순한 텍스트 검색을 넘어 **하이브리드 검색**을 지원합니다.

*   **BM25 (키워드 매칭)**: 정확한 고유 명사나 날짜 검색에 유리합니다.
*   **Vector Search (의미론적 검색)**: 사용자가 정확한 단어를 기억하지 못해도 유사한 맥락을 찾아냅니다.

`sqlite-vec`을 활용하여 로컬 환경에서도 빠른 검색 속도를 보장하며, `extraPaths` 설정을 통해 Obsidian과 같은 외부 지식 베이스를 OpenClaw의 뇌로 즉각 확장할 수 있습니다.

### 3. Agentic Memory Flush

대화가 길어져 컨텍스트 윈도우(Context Window)가 임계치에 도달하면, OpenClaw는 중요한 정보를 자동으로 파일에 기록(Flush)하고 메모리를 비웁니다. 이는 AI가 스스로 자신의 인지 자원을 관리하는 능동적인 메커니즘을 보여줍니다.

## 결론: 사용자가 통제하는 AI의 기억

OpenClaw의 메모리 시스템은 기술적인 효율성뿐만 아니라 '사용자 주권'에 초점을 맞추고 있습니다. AI의 기억은 투명해야 하며, 언제든 사용자가 마크다운 편집기로 수정할 수 있어야 한다는 점이 OpenClaw를 다른 AI 에이전트와 차별화하는 핵심 요소입니다.
